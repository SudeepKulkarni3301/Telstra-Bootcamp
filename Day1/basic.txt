Git is a popular version control system, and GitHub is a web-based hosting service for Git repositories. Here is a list of some commonly used Git and GitHub commands:

Git Commands:

git init: Initializes a new Git repository in the current directory.

git clone <repository_url>: Clones (downloads) a remote repository to your local machine.

git add <file>: Adds a file or changes to the staging area for the next commit.

git commit -m "Commit message": Creates a new commit with the changes in the staging area.

git status: Shows the status of the working directory and the staging area.

git log: Displays the commit history.

git branch: Lists all branches in the repository.

git checkout <branch_name>: Switches to the specified branch.

git merge <branch_name>: Merges changes from the specified branch into the current branch.

git pull: Fetches and merges changes from the remote repository to the current branch.

git push: Pushes local commits to the remote repository.

GitHub Commands:

git remote add origin <repository_url>: Links your local repository to a remote GitHub repository.

git push -u origin <branch_name>: Pushes the local branch to the remote repository for the first time.

git pull origin <branch_name>: Fetches and merges changes from the remote repository to the current branch.

git clone <repository_url>: Clones a remote GitHub repository to your local machine.

git fork: Creates a fork of a repository on GitHub.

git pull-request: Initiates a pull request on GitHub.

git fetch: Downloads objects and refs from another repository.

git branch -d <branch_name>: Deletes a branch (locally).

git push origin --delete <branch_name>: Deletes a branch on the remote repository.

git stash: Temporarily saves changes in a stash and reverts the working directory to the last commit.
